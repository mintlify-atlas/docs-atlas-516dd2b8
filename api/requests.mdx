## Overview

Request types represent messages sent from clients to agents that expect a response. Each request has a corresponding response type.

## Initialize

Establishes the connection with a client and negotiates protocol capabilities.

### InitializeRequest

<ParamField path="protocolVersion" type="int" required>
  The protocol version the client supports
</ParamField>

<ParamField path="clientInfo" type="Implementation">
  Information about the client implementation
  
  <Expandable title="Implementation fields">
    <ResponseField name="name" type="string" required>
      The name of the implementation
    </ResponseField>
    
    <ResponseField name="version" type="string" required>
      The version of the implementation
    </ResponseField>
    
    <ResponseField name="title" type="string">
      A human-readable title for the implementation
    </ResponseField>
    
    <ResponseField name="_meta" type="Map<String, dynamic>">
      Optional metadata
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField path="clientCapabilities" type="ClientCapabilities">
  Capabilities advertised by the client
  
  <Expandable title="ClientCapabilities fields">
    <ResponseField name="terminal" type="bool" default="false">
      Whether the client supports terminal operations
    </ResponseField>
    
    <ResponseField name="fs" type="FileSystemCapability">
      File system capabilities
      
      <Expandable title="FileSystemCapability fields">
        <ResponseField name="readTextFile" type="bool" default="false">
          Whether the client supports reading text files
        </ResponseField>
        
        <ResponseField name="writeTextFile" type="bool" default="false">
          Whether the client supports writing text files
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField path="_meta" type="Map<String, dynamic>">
  Optional metadata for the request
</ParamField>

## Session Management

### NewSessionRequest

Creates a new conversation session with the agent.

<ParamField path="cwd" type="String" required>
  The current working directory for the session
</ParamField>

<ParamField path="mcpServers" type="List<McpServerBase>" required>
  List of MCP servers to connect to. Can be `HttpMcpServer`, `SseMcpServer`, or `StdioMcpServer`
</ParamField>

<ParamField path="_meta" type="Map<String, dynamic>">
  Optional metadata
</ParamField>

### LoadSessionRequest

Loads an existing session to resume a previous conversation.

<ParamField path="sessionId" type="String" required>
  The ID of the session to load
</ParamField>

<ParamField path="cwd" type="String" required>
  The current working directory for the session
</ParamField>

<ParamField path="mcpServers" type="List<McpServerBase>" required>
  List of MCP servers to connect to
</ParamField>

<ParamField path="_meta" type="Map<String, dynamic>">
  Optional metadata
</ParamField>

## Permissions

### RequestPermissionRequest

Requests permission from the user for a tool call operation.

<ParamField path="sessionId" type="String" required>
  The ID of the session
</ParamField>

<ParamField path="options" type="List<PermissionOption>" required>
  Available permission options to present to the user
  
  <Expandable title="PermissionOption fields">
    <ResponseField name="optionId" type="String" required>
      Unique identifier for this option
    </ResponseField>
    
    <ResponseField name="name" type="String" required>
      Display name for the option
    </ResponseField>
    
    <ResponseField name="kind" type="PermissionOptionKind" required>
      The kind of permission: `allow_once`, `allow_always`, `reject_once`, or `reject_always`
    </ResponseField>
  </Expandable>
</ParamField>

<ParamField path="toolCall" type="ToolCallUpdate" required>
  Information about the tool call requesting permission
</ParamField>

<ParamField path="_meta" type="Map<String, dynamic>">
  Optional metadata
</ParamField>

## MCP Server Types

### HttpMcpServer

HTTP-based MCP server configuration.

<ParamField path="type" type="String" default="http">
  Server type (always "http")
</ParamField>

<ParamField path="name" type="String" required>
  Name of the MCP server
</ParamField>

<ParamField path="url" type="String" required>
  The HTTP URL for the server
</ParamField>

<ParamField path="headers" type="List<HttpHeader>" required>
  HTTP headers to include with requests
  
  <Expandable title="HttpHeader fields">
    <ResponseField name="name" type="String" required>
      The header name
    </ResponseField>
    
    <ResponseField name="value" type="String" required>
      The header value
    </ResponseField>
  </Expandable>
</ParamField>

### SseMcpServer

Server-Sent Events based MCP server configuration.

<ParamField path="type" type="String" default="sse">
  Server type (always "sse")
</ParamField>

<ParamField path="name" type="String" required>
  Name of the MCP server
</ParamField>

<ParamField path="url" type="String" required>
  The SSE URL for the server
</ParamField>

<ParamField path="headers" type="List<HttpHeader>" required>
  HTTP headers to include with the SSE connection
</ParamField>

### StdioMcpServer

Stdio-based MCP server configuration.

<ParamField path="name" type="String" required>
  Name of the MCP server
</ParamField>

<ParamField path="command" type="String" required>
  The command to execute
</ParamField>

<ParamField path="args" type="List<String>" required>
  Arguments for the command
</ParamField>

<ParamField path="env" type="List<EnvVariable>" required>
  Environment variables for the command
</ParamField>
